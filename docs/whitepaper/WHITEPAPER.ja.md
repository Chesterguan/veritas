## 1. ビジョン

VERITAS は、規制された環境で動作する AI エージェントのための、決定論的・ポリシー準拠・監査可能・検証可能な実行ランタイムです。本システムは、自律性や不透明な知性よりも、信頼・制御・証拠を優先します。

VERITAS は、アプリケーションや自動化製品としてではなく、基盤となる実行レイヤーとして設計されています。

## 2. 背景と動機

現代の AI エージェントは強力ですが、規制された環境では根本的に信頼性が低い場合があります。その動作はしばしば非決定論的で、監査不能であり、検証が困難です。

VERITAS は、すべての決定・アクション・出力がトレース可能かつポリシーで制約された、制御された実行環境を提供することで、これらの課題に対処します。

## 3. 設計哲学

VERITAS のすべての設計判断を規定する 10 の原則:

1. **自律性よりも制御**
2. **知性よりも証拠**
3. **創発よりも決定論**
4. **デフォルト拒否**
5. **ケイパビリティベースのセキュリティ**
6. **最小限の信頼済みコンピューティングベース**
7. **設計による監査可能性**
8. **検証可能な実行**
9. **人間による上書きは常に可能**
10. **データモデル非依存性**

## 4. ZeroClaw の設計系譜

VERITAS は、ZeroClaw の軽量かつ決定論的な設計思想を基盤としています。ZeroClaw は、最小限のエージェントカーネル・明示的な実行フロー・コンポーザビリティ・小さな信頼済みコンピューティングベースを重視します。

VERITAS はこの設計思想を継承しつつ、軽量かつモジュラーな設計を維持しながら、ポリシー強制・監査可能性・検証・安全な実行境界を導入することで拡張します。

## 5. システムモデル

VERITAS におけるエージェント実行は、制御されたケイパビリティ上で動作する決定論的なステートマシンとしてモデル化されています。

### 実行ループ

```
State → Policy → Capability → Audit → Verify → Next State
```

各トランジションは明示的であり、ポリシーチェック・監査・検証が行われた上で、エージェントが次の状態へ進みます。

## 6. トラストモデル

VERITAS における信頼は、決定論的な実行・不変の監査証跡・明示的なポリシー決定・検証可能な出力から導出されます。

本システムは、LLM の推論・外部ツール・入力データ・実行環境を固有には信頼しません。

### トラスト境界

| 信頼済み | 非信頼 |
|---------|-----------|
| ランタイムコア | LLM |
| ポリシーエンジン | ツール |
| 監査エンジン | 入力データ |
| 検証器 | 外部環境 |

## 7. ケイパビリティモデル

ケイパビリティとは、明示的なスキーマ・権限・副作用の宣言を持つ、制約されたツールです。

外部世界とのすべてのやり取りは、ポリシー制御のもとでケイパビリティを通じて行われなければなりません。

## 8. ポリシーとガバナンス

VERITAS はデフォルト拒否の実行を強制します。ポリシー決定は、サブジェクト・アクション・リソース・コンテキストを評価し、以下の 3 つの結果のいずれかを決定します:

- **許可 (Allow)**
- **拒否 (Deny)**
- **承認要求 (Require Approval)**

ポリシーは決定論的・説明可能・監査可能です。

## 9. 監査とトレーサビリティ

すべての実行イベントは、検証可能な実行グラフを形成する追記専用のイベントストリームに記録されます。各イベントには以下が含まれます:

- 状態遷移
- ケイパビリティ呼び出し
- ポリシー決定
- 検証結果

本システムは、再現可能かつ改ざん防止された実行トレースをサポートします。

## 10. 検証モデル

すべての出力は、以下を含む検証チェックに合格する必要があります:

- スキーマ検証
- ルール検証
- リスク評価

機密性の高い操作については、オプションの二次検証と人間によるレビューが必要となる場合があります。

## 11. セキュリティモデル

VERITAS は最小権限・隔離されたケイパビリティ実行・厳格な境界制御を強制します。

ランタイムは直接のシステムアクセスを許可せず、すべての外部コンポーネントを非信頼とみなします。

## 12. データモデル非依存性

VERITAS のコアランタイムは、FHIR・OMOP・独自スキーマなど、特定のヘルスケアまたはエンタープライズデータモデルから独立しています。

ドメイン固有のアダプターは、ケイパビリティを通じて外部に実装されます。

## 13. 拡張性

VERITAS は以下のための標準化されたインターフェースを提供します:

- ケイパビリティ
- ポリシーエンジン
- 監査ストレージ
- 検証モジュール

外部の貢献者は、信頼済みコアを変更することなくシステムを拡張できます。
