## 1. 愿景 {#1-vision}

VERITAS 是一个确定性、策略绑定、可审计、可验证的执行运行时，专为在受监管环境中运行的 AI 智能体设计。该系统将信任、控制和证据置于自主性和不透明智能之上。

VERITAS 被设计为基础执行层，而非应用产品或自动化产品。

## 2. 动机 {#2-motivation}

现代 AI 智能体功能强大，但在受监管环境中从根本上是不可靠的。其行为往往具有非确定性、不可审计，且难以验证。

VERITAS 通过提供受控执行环境来解决这些局限性，在该环境中每一个决策、行动和输出均可追溯并受策略约束。

## 3. 设计哲学 {#3-design-philosophy}

以下十条原则统领所有 VERITAS 设计决策：

1. **控制优先于自主**
2. **证据优先于智能**
3. **确定性优先于涌现**
4. **默认拒绝**
5. **基于能力的安全**
6. **最小可信计算基**
7. **可审计性内建于设计**
8. **可验证执行**
9. **人工干预始终可行**
10. **数据模型独立性**

## 4. ZeroClaw 设计传承 {#4-zeroclaw-design-lineage}

VERITAS 建立在 ZeroClaw 轻量级与确定性哲学之上。ZeroClaw 强调最小化智能体内核、显式执行流、可组合性以及小型可信计算基。

VERITAS 在保留轻量级与模块化设计的同时，通过引入策略执行、可审计性、验证和安全执行边界对这一哲学进行了扩展。

## 5. 系统模型 {#5-system-model}

VERITAS 中的智能体执行被建模为在受控能力上运行的确定性状态机。

### 执行循环

```
状态 → 策略 → 能力 → 审计 → 验证 → 下一状态
```

每次状态转换均是显式的、经过策略检查的、已审计的，并在智能体进入下一状态前完成验证。

## 6. 信任模型 {#6-trust-model}

VERITAS 中的信任来源于确定性执行、不可变审计跟踪、显式策略决策和可验证输出。

系统不会固有地信任 LLM 推理、外部工具、输入数据或执行环境。

### 信任边界

| 可信 | 不可信 |
|---------|-----------|
| 运行时核心 | LLM |
| 策略引擎 | 工具 |
| 审计引擎 | 输入数据 |
| 验证器 | 外部环境 |

## 7. 能力模型 {#7-capability-model}

能力代表具有显式模式、权限和副作用声明的受约束工具。

与外部世界的所有交互必须通过处于策略控制下的能力进行。

## 8. 策略与治理 {#8-policy-and-governance}

VERITAS 强制执行默认拒绝。策略决策通过评估主体、操作、资源和上下文，得出以下三种结果之一：

- **允许**
- **拒绝**
- **需要审批**

策略是确定性的、可解释的且可审计的。

## 9. 审计与可追溯性 {#9-audit-and-traceability}

所有执行事件均记录在一个仅追加的事件流中，形成可验证的执行图。每个事件包含：

- 状态转换
- 能力调用
- 策略决策
- 验证结果

系统支持可重放和防篡改的执行追踪。

## 10. 验证模型 {#10-verification-model}

所有输出必须通过以下验证检查：

- 模式验证
- 规则验证
- 风险评估

对于敏感操作，可能需要可选的二次验证和人工审查。

## 11. 安全模型 {#11-security-model}

VERITAS 强制执行最小权限原则、隔离的能力执行和严格的边界控制。

运行时不允许直接访问系统，并假设所有外部组件均不可信。

## 12. 数据模型独立性 {#12-data-model-independence}

VERITAS 核心运行时独立于特定的医疗健康或企业数据模型，例如 FHIR、OMOP 或专有模式。

领域专用适配器通过能力在外部实现。

## 13. 可扩展性 {#13-extensibility}

VERITAS 为以下组件提供标准化接口：

- 能力
- 策略引擎
- 审计存储
- 验证模块

外部贡献者可以在不修改可信核心的情况下扩展系统。
